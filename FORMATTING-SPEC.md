# HILLHEAD MIXED — SESSION PLAN FORMATTING SPEC
# Give this document to Claude alongside any new session plan to ensure consistent formatting.

---

## WHAT TO DO WITH THIS DOCUMENT

When given a new session plan (as a Google Doc, .docx, or pasted text), use this spec sheet
to convert it into a new HTML page that matches the existing site exactly.
Do not invent new styles or components. Only use the elements defined below.

---

## 1. FILE NAMING

Name the output HTML file using this format:
  session-DD-month.html
  e.g. session-19-april.html, session-03-may.html

---

## 2. PAGE STRUCTURE

Every session page must follow this exact structure, top to bottom:

  1. <head> — link to style.css, copy inline <style> block from existing session page
  2. <nav> — site navigation bar (copy exactly from any existing session page)
  3. .page-banner.banner-blue — session date as h1, subtitle with duration and start time
  4. .container > .session-layout — all content goes inside here
  5. Session Aims block
  6. Section divider <hr>
  7. Timeline rows (early session: warm up etc.)
  8. Section divider <hr>
  9. Main drill / focus content
  10. Break badge (if applicable)
  11. Further drills or content
  12. Section divider <hr>
  13. Games block (if applicable)
  14. Drill Explanation section with embedded video (if applicable)
  15. Collapsible section for any optional/extra reading (if applicable)

---

## 3. ELEMENT RULES

### Page Banner
- h1 = date only (e.g. "22 March")
- subtitle p = "SESSION THEME — X hours, X:XX PM"
- Always use class="page-banner banner-blue"
- Breadcrumb: Home / Training Plans / [Date]

### Session Aims
- Always use the .aims-block with .block-label blue and .aims-list
- Numbers are auto-generated by CSS — just use <li> items
- Maximum 4 aims per session

### Timeline Rows
- Use .time-row for every scheduled time slot
- time-stamp = time only (e.g. "3:00", "3:15")
- h3 = activity name
- p = optional short description (omit if nothing to add)
- If a time row introduces a major section below it (e.g. Zone, Endzone),
  add style="border-bottom:none;" to that row

### Section Dividers
- Use <hr class="section-divider"/> between major sections
- Do NOT put a divider between every single time row — only between major blocks

### Offence & Defence Cards
- Use .zone-grid with two .zone-card divs inside
- Offence card: class="zone-card offence" — blue top border, blue h3
- Defence card: class="zone-card defence" — green top border, green h3
- Use .bullet-list inside each card
- If there is only offence OR only defence content, use a single .content-card.blue-top
  or .content-card.green-top instead (full width)

### Bullet Lists
- Always use <ul class="bullet-list"> with <li> items
- For sub-points under a bullet, use <ul class="sub-list"> nested inside the <li>
- Never use default HTML list styles

### Numbered Guidelines
- Use <ol class="numbered-list"> for any numbered coaching guidelines
- These auto-style with blue numbers

### Break Badge
- Use for any rest/water break in the schedule
- Include time range and label: "⏸ &nbsp; 4:10 – 4:15 &nbsp; Break"
- If the break has an activity (e.g. "tell someone your frisbee journey") include it in the text

### Games Block
- Always use .games-block with green left border
- .block-label green for the heading ("Game Format")
- Each rule gets its own .game-rule row with a .rule-tag
- Use rule-tag.blue for rules, rule-tag.green for format
- Always end with .win-note containing a trophy emoji and win condition

### Images
- All images go in the /media/ folder
- Reference as src="media/filename.png"
- Always use class="drill-image" for full-width styled images
- Add class="image-caption" <p> underneath if the image needs a label
- Place images in the logical position in the document where they appeared in the source

### Embedded Videos
- Section heading: <h3>Drill Explanation</h3>
- Sub-label above video: <div class="block-label blue">Drill Name</div>
- Video goes inside <div class="iframe-wrap">
- Ultiplays embed: use the embed URL from the share link
- YouTube embed: use youtube.com/embed/VIDEOID format, add ?start=SECONDS if linking to a timestamp
- Always set width="100%" — never fixed pixel widths

### Collapsible Sections
- Use for any content marked as optional, extra reading, or coach's notes
- Title the summary clearly (e.g. "Inside Joel's Mind", "Coach's Notes")
- Add a brief disclaimer in a <span class="disclaimer"> if appropriate
- Body content uses <div class="mind-content"> with <div class="mind-section"> per topic
- Each mind-section has an h3 heading and p paragraphs

### Drill Links (Ultiplays / external)
- If the source doc contains a URL to an external drill, render it as:
  <a href="URL" target="_blank" class="drill-link">▶ View Drill on Ultiplays</a>
- Place it directly below the time-row h3 it relates to

---

## 4. COLOUR RULES

Blue (#1e6fbf) = Offence, O-line content, general session info
Green (#1a9e5c) = Defence, D-line content, games/results

Apply consistently:
- Offence card header → blue
- Defence card header → green
- O Line aims column label → blue
- D Line aims column label → green
- Games block border → green
- General session aims and timeline → blue
- rule-tag on rules → blue
- rule-tag on format → green

---

## 5. TRAINING INDEX UPDATE

After creating the new session HTML file, also update training.html:
- Add a new .plan-card block above the instructions comment
- href points to the new session file
- .plan-date shows the day number and 3-letter month
- .plan-info h3 = short session theme title (3-5 words)
- Tags: use tag-blue for offensive/general themes, tag-green for defensive themes, tag-gray for duration and time
- Update the .plan-count-badge number at the top

---

## 6. THINGS TO NEVER DO

- Never use inline styles for colours — always use var(--blue) or var(--green)
- Never use <b> or <i> tags — use <strong> or <em>
- Never use default bullet points (list-style) — always use .bullet-list or .numbered-list
- Never create new CSS classes not already in style.css
- Never use fixed pixel widths on iframes — always width="100%"
- Never put a divider immediately after the page banner
- Never skip the breadcrumb in the page banner
- Never put the session aims inside a time-row
